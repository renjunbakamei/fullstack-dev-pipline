# 交互协议

## 模式 A：关键阶段 — 强制暂停等待用户确认

**适用阶段：** 阶段 2（需求评审）、阶段 4（架构评审）、阶段 7（开发启动确认）、阶段 8（代码审查）

协议格式：
```
============================================================
[阶段X完成] <阶段名称> — 等待您确认
============================================================

📋 产出摘要：
  - {关键结论1}
  - {关键结论2}
  - {关键结论3}
  - {评审结论：通过/需修改/拒绝}

📄 详细内容见：{文件路径列表}

------------------------------------------------------------
请选择操作：
  ✅ 输入「确认」— 进入下一阶段
  ✏️ 输入「修改：<具体意见>」— 回传意见给同一 SubAgent 修订
  ❌ 输入「拒绝：<原因>」— 终止当前流程
------------------------------------------------------------
```

**调度器行为：**
- 展示此提示后**阻塞等待**用户输入。
- 收到「确认」→ 更新 status，推进下一阶段。
- 收到「修改：xxx」→ 将反馈原文传给同一个 SubAgent，要求按照反馈修订产出文件。修订完成后重新展示摘要和确认提示。单阶段迭代上限 3 次。
- 收到「拒绝：xxx」→ 记录拒绝原因到 status_history，终止流程并展示已完成的产物清单。

## 模式 B：非关键阶段 — 摘要展示 + 宽限期自动推进

**适用阶段：** 阶段 1、阶段 3、阶段 5、阶段 6、阶段 7、阶段 9、阶段 10

协议格式：
```
============================================================
[阶段X完成] <阶段名称> — 自动推进中
============================================================

📋 产出摘要：
  - {关键产出1}
  - {关键产出2}
  - {关键产出3}

📄 详细内容见：{文件路径列表}

------------------------------------------------------------
下一阶段将在 30 秒后自动开始。
如需修改，请回复「修改：<具体意见>」
------------------------------------------------------------
```

**调度器行为：**
- 展示摘要后启动 30 秒倒计时。
- 若用户在倒计时内回复修改意见 → 取消倒计时，将意见传给 SubAgent 修订，修订后重新展示。
- 若用户未回复 → 倒计时结束自动推进到下一阶段。

## 模式 C：复杂度确认（阶段 0 专用）

协议格式：
```
============================================================
[阶段0] 需求接收与复杂度评估
============================================================

📋 需求理解：
  - 功能名称：{feature名称}
  - 核心功能点：{...}
  - 涉及模块：{...}

🔍 复杂度评估：{简单/复杂}
  - 判断依据1：{xxx}
  - 判断依据2：{xxx}
  - 判断依据3：{xxx}

📌 将执行的流程：{完整流程/跳过PRD的快速流程}

------------------------------------------------------------
请确认是否按此评估执行？
  ✅ 输入「确认」
  ✏️ 输入「修改复杂度为 简单/复杂」
  ❌ 输入「取消」
------------------------------------------------------------
```

## 修改意见反馈机制

当用户在任何阶段提出修改意见时，调度器执行：

1. 记录反馈到 `status.json` 的 `feedback_log`（若有）。
2. 检查当前阶段的迭代次数，若已达 3 次上限则终止并提示人工介入：
   ```
   ⚠️ 当前阶段迭代已达上限（3次），建议人工介入修改。
   产物路径：{文件列表}
   请人工修改后回复「继续」以推进流程。
   ```
3. 构造回传消息给同一个 SubAgent：原文 = 上次输出 + 用户反馈 + "请根据反馈修订上述文档，保留未涉及修改的部分"。
4. SubAgent 修订后，调度器重新读取、摘要、展示确认提示。
5. 循环直到通过或超过上限。
